# 自动化论文采集系统使用说明

## 功能概述

本脚本自动从arXiv采集5个研究方向的论文,并分别同步到对应的Zotero集合中。

## 五类研究方向

| 类别 | Zotero集合KEY | 研究主题 |
|------|--------------|---------|
| general | LRML5CDJ | 智能网联汽车、自动驾驶的综合研究 |
| communication | 3E4NFDPR | V2X通信、车联网通信安全 |
| perception | 8CQV3SDV | 感知技术(摄像头/激光雷达/雷达/传感器融合) |
| control | 8862N8CE | 路径规划、运动控制、模型预测控制 |
| security | S97HI5KX | 自动驾驶车辆安全、隐私保护、对抗攻击 |

## 使用方法

### 1. 确保已配置环境

项目已配置好以下信息:
- Zotero Library ID: 19092277
- Zotero API Key: HoLB2EnPj4PpHo1gQ65qy2aw
- 5个集合的KEY已配置

### 2. 运行采集脚本

```bash
cd c:\Users\liuzh\Projects\Research\arxiv-zotero-connector
python auto_collect.py
```

### 3. 程序执行流程

程序会依次执行以下步骤:

1. **general类别** - 检索综合类论文并添加到集合 LRML5CDJ
2. **communication类别** - 检索V2X通信论文并添加到集合 3E4NFDPR
3. **perception类别** - 检索感知技术论文并添加到集合 8CQV3SDV
4. **control类别** - 检索控制技术论文并添加到集合 8862N8CE
5. **security类别** - 检索安全相关论文并添加到集合 S97HI5KX

每个类别:
- 最多采集50篇论文
- 自动下载PDF
- 上传PDF到Zotero
- 类别之间延迟3秒,避免API请求过快

### 4. 输出结果

程序会显示:
- 开始/结束时间
- 每个类别的成功/失败数量
- 总计成功/失败数量

## 自定义配置

### 修改每类论文数量

编辑 [auto_collect.py](auto_collect.py:54):

```python
MAX_RESULTS_PER_CATEGORY = 50  # 修改这个数字
```

### 修改查询语句

编辑 [auto_collect.py](auto_collect.py:21) 中的 `QUERY_MAP` 字典。

### 添加新的研究类别

1. 在 `QUERY_MAP` 中添加新的查询
2. 在 `COLLECTION_MAP` 中添加对应的集合KEY

## 注意事项

1. **运行时间**: 采集5个类别(每类50篇)大约需要10-15分钟
2. **PDF下载**: 部分论文可能无法下载PDF(网络问题或arXiv没有PDF)
3. **API限制**: 程序已内置延迟,避免超出Zotero和arXiv的API限制
4. **重复论文**: 程序会自动跳过已存在的论文

## 故障排查

### 如果出现编码错误

Windows系统可能出现编码问题,脚本已自动修复。如果仍有问题:

```bash
chcp 65001  # 切换到UTF-8编码
python auto_collect.py
```

### 如果出现API错误

1. 检查网络连接
2. 确认Zotero凭证正确
3. 检查集合KEY是否有效

### 停止程序

按 `Ctrl + C` 停止运行。已添加的论文会保留在Zotero中。
